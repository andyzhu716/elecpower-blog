# Hugo Website Transformation Plan: Only English + Structure Optimization + Visual Enhancement

## Project Goal

Transform current bilingual Hugo site into an English-only professional content site with:
1. Remove all Chinese content and bilingual configuration
2. Upgrade from basic corporate site to professional content + product capability site
3. Optimize homepage, Blog, FAQ, and internal linking structure
4. Improve visual consistency, professionalism, and unity
5. Maintain Hugo build, Schema output, and GitHub Pages deployment stability

## Constraint Checklist

- [ ] Main site must be English-only
- [ ] No Chinese navigation, buttons, homepage copy, or metadata residues
- [ ] Do NOT break schema, CNAME, GitHub Actions workflow, or deployment pipeline
- [ ] Do NOT refactor complex schema or deployment files in this round

## Risk Categories

### Low Risk Changes
- Copy translation to English
- Button label updates
- Navigation label updates
- Homepage module reordering
- FAQ copy additions
- Card style unification
- Spacing/typography adjustments

### Medium Risk Changes
- Deleting Chinese pages
- Changing URLs/slugs
- Adding new sections
- Homepage template structure adjustments
- Adding related content modules

### High Risk Changes (NOT in scope for this round)
- Complex Hugo template logic modifications
- Schema/microformat refactoring
- Deployment workflow changes
- CNAME changes
- Major theme layout overhauls
